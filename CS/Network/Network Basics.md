# 1. 처리량과 지연시간

## 1.1 처리량(Throughput)

단위 시간 동안 실제로 전송되는 데이터의 양얼마나 많은 데이터를 전송할 수 있는가를 나타내는 지표

### 단위: bps

### SI 접두어

!image.png

### 처리량에 영향을 주는 요소

### 1. 대역폭(Bandwidth)

전송 가능한 최대 데이터 양

- 대역폭 ↑ → 처리량 ↑

### 2. 패킷 손실(Packet Loss)

패킷이 손실되면 재전송이 발생한다.

→ 처리량 감소

### 3. 네트워크 혼잡(Congestion)

사용자가 많아질수록

- 충돌 증가
- 대기시간 증가
- 처리량 감소

### 4. 하드웨어 성능

- CPU
- Memory
- NIC(Network Interface Card)
- Switch

성능이 낮으면 처리량도 감소한다.

### 특징

- 높을수록 좋다.
- 다운로드 속도와 밀접하다.
- 서버가 동시에 처리 가능한 양과 관련 있다.

---

## 1.2 지연시간(Latency)

데이터가 목적지까지 도착하는 데 걸리는 시간

### 단위: ms

### 지연시간 구성 요소

#### 1. 전송 지연 (Transmission Delay)

- 패킷을 네트워크에 올리는 시간

```
패킷 크기
────────────
전송 속도
```

#### 2. 전파 지연 (Propagation Delay)

- 신호가 실제 이동하는 시간
- 거리와 비례한다.
- 서울 → 부산 < 서울 → 미국

#### 3. 처리 지연 (Processing Delay)

라우터가

- IP 확인
- 목적지 확인
- 라우팅 결정

하는 시간

#### 4. 큐잉 지연 (Queuing Delay)

- 패킷이 많으면 Queue에서 대기
- 트래픽이 많을수록 증가한다.

### 특징

- 낮을수록 좋다.
- 게임, 화상회의 등에 매우 중요하다.

## 처리량과 지연시간 비교

| 구분 | 처리량 | 지연시간 |
| --- | --- | --- |
| 의미 | 얼마나 많이 보내는가 | 얼마나 빨리 도착하는가 |
| 단위 | Mbps, Gbps | ms |
| 좋은 상태 | 높다 | 낮다 |
| 영향 | 다운로드 속도 | 응답속도 |

### ⭐ 면접 핵심

**Q. 처리량과 대역폭의 차이**

A. 대역폭은 이론적인 최대 전송 속도이고, 처리량은 실제 네트워크 환경에서 측정되는 전송 속도이다.

# 2. 네트워크 토폴로지와 병목현상

## 2.1 네트워크 토폴로지(Topology)

네트워크 장치들이 서로 연결된 구조

토폴로지에 따라 성능/장애 대응/확장성이 달라진다.

!image.png

## 1. 버스(Bus)

### 장점

- 설치 비용 저렴
- 구조 단순

### 단점

- 메인 케이블 장애 시 전체 장애
- 충돌 발생 가능

## 2. 스타(Star)

### 장점

- 가장 많이 사용
- 관리 쉬움
- 장애 원인 파악 쉬움

### 단점

- 중앙 Switch 장애 시 전체 네트워크 마비

## 3. 링(Ring)

### 장점

- 충돌 적음

### 단점

- 하나의 장애가 전체에 영향

## 4. 메시(Mesh)

모든 노드가 서로 연결

### 장점

- 장애에 매우 강함
- 신뢰성 높음

### 단점

- 비용 매우 높음

## 5. 트리(Tree)

### 장점

- 확장성 좋음

### 단점

- 상위 장비 장애 영향 큼

# 2.2 병목현상

전체 시스템 성능이 가장 느린 부분 때문에 제한되는 현상

### 원인

- CPU 부족
- 메모리 부족
- 네트워크 대역폭 부족
- 디스크 I/O
- DB 성능

### 해결 방법

- Scale Out
- Scale Up
- 캐시 적용
- DB 튜닝
- 로드 밸런싱

### ⭐ 면접 핵심

Q. 병목현상이란?

A. 시스템 전체 성능이 특정 자원의 성능 한계 때문에 제한되는 현상

# 3. 네트워크 분류

네트워크는 범위에 따라 분류된다.

| 종류 | 범위 | 예시 |
| --- | --- | --- |
| PAN | 개인 | Bluetooth |
| LAN | 건물 | 회사, 학교 |
| MAN | 도시 | 도시 통신망 |
| WAN | 국가 | 인터넷 |

## PAN

Personal Area Network

- Bluetooth
- NFC

## LAN

Local Area Network

- 회사
- 학교
- 집

특징

- 빠른 속도
- 낮은 지연시간

## MAN

Metropolitan Area Network

도시 규모

LAN 여러 개 연결

## WAN

Wide Area Network

국가 규모

대표 예시

- 인터넷

특징

- 지연시간이 큼

# 4. 네트워크 성능 분석 명령어

## ping

### 목적

상대 서버와 연결 가능한지 확인

### 확인 가능한 것

- RTT(Round Trip Time): 패킷이 출발지에서 목적지까지 갔다가, 다시 출발지로 돌아오는 데 걸리는 왕복 시간
- Packet Loss

```
ping google.com
```

## traceroute (Windows : tracert)

### 목적

목적지까지 거치는 라우터 확인

```
traceroute google.com
```

확인 가능

- 어느 구간에서 느린지
- 어느 라우터에서 장애가 발생했는지

## netstat

현재 네트워크 상태 확인

확인 가능

- 열린 포트
- TCP 연결
- 라우팅 정보

```
netstat -an
```

## nslookup

DNS 조회

```
nslookup google.com
```

도메인을 IP로 변환하는지 확인

## arp

ARP Cache 확인

```
arp -a
```

## tcpdump

패킷 캡처

Linux에서 사용

```
tcpdump
```

## Wireshark

GUI 기반 패킷 분석 프로그램

실무에서 가장 많이 사용된다.

### ⭐ 면접 핵심

**ping과 traceroute 차이**

| ping | traceroute |
| --- | --- |
| 연결 확인 | 경로 확인 |
| RTT 확인 | 라우터 확인 |

# 5. 네트워크 프로토콜 표준화

## 프로토콜(Protocol)

컴퓨터들이 통신하기 위한 규칙

## 표준화가 필요한 이유

제조사가 달라도 동일한 규칙을 사용하면 통신 가능

예)

- Windows ↔ Linux
- Cisco ↔ Juniper
- Chrome ↔ Nginx

## 대표적인 표준화 기관

| 기관 | 역할 |
| --- | --- |
| IETF | 인터넷 프로토콜(RFC) 표준 제정 |
| IEEE | Ethernet, Wi-Fi(802 시리즈) 표준 |
| ISO | OSI 7계층 국제 표준 |
| ITU-T | 국제 통신 표준 제정 |
